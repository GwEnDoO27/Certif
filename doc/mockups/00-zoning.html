Zoning fonctionnel — 5 pages

Stade 0 du processus design : Zoning → Wireframe → Maquette → Mockup HD. Découpage en zones fonctionnelles avant tout choix graphique. Chaque couleur représente le rôle UX d'une zone, pas son apparence finale.

Navigation
Header, menu, accès
Identité / Hero
Logo, titre, branding
Contenu principal
Info clé, listes, données
Action / CTA
Bouton primaire, conversion
Auxiliaire
Sidebar, infos complémentaires
Saisie
Formulaire, input, upload
Feedback
Erreur, alerte, état système
Méta / Footer
Copyright, légal
1 — Login/
Identité / Hero
Logo + Titre "Bienvenue" + Accroche
Premier contact visuel
Feedback (conditionnel)
Message d'erreur si login échoue
Saisie
Formulaire 2 champs : email + mot de passe
Validation HTML5 native + ARIA labels
Action principale
CTA "Se connecter" → POST /sys/login
Méta
Copyright
Objectif fonctionnel
Page d'entrée. Friction minimale : 4 zones + 1 conditionnelle (feedback erreur). Aucune navigation parasite (pas de header).
2 — Home/home
Navigation globale (logo, theme, admin*, logout)
Widget contextuel
Horloge live + date
Accueil personnalisé
"Bon retour !" — accroche
Contenu principal
Sélecteur d'applications groupées
8 accents colorés rotatifs par carte · Chargement dynamique selon rôle utilisateur
Auxiliaire
Fil d'actualités
News système, mises à jour
Objectif fonctionnel
Tableau de bord post-login. 2/3 contenu principal, 1/3 auxiliaire. Header sticky pour navigation persistante. Widgets de contexte en haut pour réduire la charge cognitive avant l'action.
3 — Admin · Onglet Utilisateurs/admin
Navigation locale : titre "Administration" + onglets segmentés (Utilisateurs · Applications · Analytics) + CTA "Nouveau"
Stat globale 1
Utilisateurs (count + badge admins)
Stat globale 2
Applications (count + badge actives)
Stat globale 3
État système (uptime)
Contenu principal
Liste CRUD utilisateurs (avec recherche + pagination)
Actions par ligne : éditer ✎ / supprimer × · Badges de rôles colorés
Auxiliaire temps réel
Panel "En ligne · N" via WebSocket
Utilisateurs connectés + route actuelle · Sticky au scroll
⊕ CTA Nouveau utilisateur (ouvre modal)
Objectif fonctionnel
Console d'administration. 4 niveaux d'information : navigation (onglets) → stats (vue d'ensemble) → contenu (action CRUD) → auxiliaire (présence temps réel). Garde d'accès via vérif verify-admin au mount.
4 — Convert · Transformation Factures MB/facture-mb/home
Identité de l'outil : icône XLSX + titre + sous-titre "Transforme les .edi en excel"
Indicateur de progression
Stepper 2 étapes : Configuration → Traitement
Saisie — colonne configuration
3 accordéons de codes (généraux, comptables, journal)
Inputs labellisés avec indicateur "modifié" · Sauvegarde par badge
Action secondaire (bleu)
CTA "Enregistrer les modifications"
Désactivé si aucune modification
Saisie — colonne fichiers
Drop zone .txt (drag&drop + click) + liste des fichiers sélectionnés
Filtre type MIME · suppression individuelle ou globale
Action principale (vert)
CTA "Lancer la transformation" → POST /facture-mb/conversion
Téléchargement direct du .xlsx généré
Onboarding
Guide rapide en 3 étapes
Aide inline pour utilisateur novice
Objectif fonctionnel
Outil métier — conversion EDI vers XLSX. 2 actions distinctes visuellement séparées : sauvegarde config (bleu) ≠ lancement traitement (vert). Le stepper rappelle le parcours en 2 temps. Guide rapide à droite pour onboarding sans documentation externe.
5 — 404 / Not Found*
Feedback critique
Identifiant erreur "404" + message rassurant
Action principale
Retour à l'accueil
Action secondaire
Page précédente
Lien contact support — escalade
Objectif fonctionnel
Page d'erreur. Ne jamais bloquer l'utilisateur : feedback clair + 2 issues + support. Conforme RGAA (aide à l'utilisateur en cas d'erreur).
